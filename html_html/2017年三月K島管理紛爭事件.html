2017年三月K島管理紛爭事件

2017年三月Ｋ島管理紛爭事件
給惡意破壞者的話：此頁早已被編輯者本人及無數島民備份，奉勸當事人無謂白費心機！
此條目有可能被有心人士再次破壞，請注意。
注意：這個條目敘述的是2017年三月，由Ｋ島管理員之間引發的事件。
關於2017年十一月由有心島民引發的類似事件，因編輯上的技術面問題[1]而暫不另外成立條目，另起段落於本條目中簡介。

因為很重要，所以請各路編輯者於編輯前一定要看過此段落

  • 因為下面原因，請有興趣的朋友在安全、合法、尊重、中立、經過整理的情況下補完本條目：
    • 本條目記載的是一個涉及管理員的事件。
    • 防止觸犯個資法
    • 本條目所敘述之事件提及之人物，於Komica的空間提供上皆有一定影響力[2]
      • 當然，本事件當事人=管理員也說過關於ＫWiki所有條目之設立，會以管理員身份抱持中立態度處置。


目錄

事件起因

2017年2月28日
  • 有鑑於多數島民反映多數版面形同棄置，Ｋ島出沒率最高的管理員、林奈（下稱Ａ氏）[3]，決心大刀闊斧進行改革。
    • 該改革被稱之為廢棄版面移置處理辦法，於二月底公告討論。（討論串：廢棄版面移置處理辦法）
    • 雖然說是改革，也只是設立一些規範標準以判別版面是否為蚊子版或管理員已無心力管理，之後將符合條件的版面移至［失效連結］處。
  • 公告一出，其他使用者便在該文回覆抱怨一些黑幕。而Ａ氏也承諾會將狀況回報給個聯絡版面的管理負責人。
    • 黑幕不外乎是數個來自網域mymoe的版面討論風氣不佳、管理也未確實，甚至是放任以及默認違規的情形，等同【管理員放任鬧版、甚至無心管理，使版面成為蚊子版】。
2017年3月6日
  • 當日，2CAT聯合站創辦人兼管理者misatocat（下稱Ｍ氏）突然於該討論串出沒，並發表你所公布的條文，我從未知悉，深感困惑、難過及不理解，反對條例。
    • 稍晚，Ａ式在另一篇公告文之下回覆發表相關版面的狀況「太離譜」「考慮移置」等文字。
  • 根據日後某自發投訴串#108的補充說明，此時應該有展開三方、甚至聯合站成員的會談。
    • 途中可能因討論過程出現摩擦，加上之前類似事件的積怨（恕本說明省略之），1883氏希望Ａ氏休息冷靜。

Ｋ島炎上

2017年3月7日
  • Ｍ氏發表站務公告，表示「管理員"林奈"正式對1883提出請辭，今後不再委任林奈對聯合站進行任何管理行為，終止一切管理職務。」
    • 收回其管理權限的同時，ＫWIKI的2cat條目同步將其代號移動至離職管理者區域。
  • 公告一出，島民一片譁然，不敢置信對大家來說管理率最高的管理員竟然主動退休，【被退休】的謠言不脛而走，後來也被本人證實為真
  • 而且，下述事件的發生更是助長島民支持Ａ氏的風氣：
    • 發出站務公告的Ｍ氏，並非在該公告內而是在四格版進行後續說明，但該後續說明與公告完全不符。
      • 四格板說法：因Ａ氏再次越權管理失職，Ｍ氏與1883氏決議拔職）[4]
      • 公告說法：Ａ氏向1883氏提出退休申請，Ｍ氏與1883核准）
    • 聲優廚於當晚七點起大量出沒於特定版面（例如GIF、角色版）達數小時，但未見管理員前去處理（島民尤其希望Ｍ氏去處理未果）
      • 因此島民認定創站以來近十年未出沒的Ｍ氏僅為了將Ａ氏被退休而出沒，因此出現了反Ｍ氏的聲浪
2017年3月8日
  • 匿名使用者發表了「投訴misatocat ◆S3571GlAtI」文章，要求站方出面給予合理解釋。各種反對對此次事件處置的聲音也隨之四起，希望Ｍ氏與1883氏收回撤職成命
    • 主要的反對理由不外乎是：［沒有稱職的管理員接任Ａ氏空缺］、［因為還在規劃中的改革計畫有越權疑慮，就將認真做事的管理員拔職，不符比例原則］
  • 之後1883氏在貓管理部發布公告，對事件進行說明（即事件說明串）。
  • Ａ氏也接著發文進行兩點聲明：
    • 舉凡退休也好、被退休也好，都是誤會一場
    • 為這次改革計畫未知會到Ｍ氏首度致歉（資訊來自某自發投訴串#108）；並首度確認該改革計畫是在只經過1883氏確認的情況下發表（但僅在提供島民討論階段）
  • 雖然有些島民認為說了跟沒說一樣，但事件與誤會已因1883氏的公告而趨緩下來；而Ａ氏的管理權限從下述各種跡象與後續討論來看已經恢復：
    • 之前被檢舉的聲優廚亂版文已消失；ＫWIKI專門回報串由Ａ氏本人確認仍是管理員，還回覆了ＫWIKI臨時發生520的原因
    • ＫWIKI的2cat條目再度將Ａ氏歸類到現任管理群並凍結之；當時經常遭刪除的小馬條目也已成功復原並凍結之。
  • 但部分島民仍有不滿，因為：
    • 管理方沒有對管理不當的版面如何處置進行說明。
    • Ｍ氏遲遲未針對於貓管公告與四格版前後不一的說法提出解釋。
2017年3月9日
  • Ａ氏在貓管理部發表【事情原委】一文，除對整個事件二度致歉外，並證實引發此事件的主因在於廢棄版面移置處理辦法僅經過1883審閱就發布討論。
    • 該辦法雖經1883氏確認大致上沒問題，但因Ａ氏認為改革不應拖延，而未採納1883氏意見【需與聯合站其他成員商量】而發表之
      • 次因是因為管理群間早有派系而林奈放話說要處理mymoe短片2的放任GIF上車串而移置版面位置招惹某派系不滿
      • 而在該串被鎖，進行深入討論期間，因發生不明爭執、使1883氏希望Ａ氏休息冷靜之對話被Ｍ氏誤解，因此發布Ａ氏退休之公告，然後引發此次炎上。
  • 另，在該文章中，Ａ氏同時表示【廢棄版面移置處理辦法】已正式提上討論行程。
  • 當然，依然有部分島民不太買帳，認定該文其實就是單純的被道歉文，如果Ｍ氏沒有出來對下面的狀況與問題點（依提出序列出）作出解釋，這次事件不算真正結束。
    1. 為何公告說法與四格版說法不符
    2. 為何Ｍ氏在公告說法與四格版說法發布後就再度下潛，不像Ａ氏與1883氏有至少發表一篇文說明
    3. 為何不先找到遞補的管理員之後再發出公告（不管今天Ａ氏是退休也好、被退休也好）
  • 中午，Ｍ氏終於出面說明事情原委。但對島民來說其實都是抱怨文，問題只是增加了而已。

重大轉折

  • 承上。在Ｍ氏與島民的說明和辯論中，Ｍ氏突然表示從未在四格版面發表過文章......
    • 突然成為羅生門？接下來請反白
  • 本來只是猜測，但不久後有島民藉由估狗搜尋指出Ｍ氏在Komica所使用的Trip碼原形可以用google找的到清單(咦?)。
    • 隨後島民竟真的發現記載大量Trip碼的網站[5]，證實因為Ｍ氏所使用的Trip碼原型其組成文字太簡單，該碼輕鬆被盜。
    • 不少島民抱著試驗性質發文，然後發文成功。當然也有島民玩過了頭。
      • 這個發現令大量島民開始體驗成為管理員的感覺，雖然不可能有實質管理權
  • 隨後Ａ氏的查證更證明了：那篇引起軒然大波的關鍵文章＝被認為是Ｍ氏發表的的［四格板說法］，其IP是國外的代理＝發文IP是外國VPN＝四格板Ｍ氏幾乎可確認是冒牌貨。該IP已鎖定。
  • 經由此事也間接指出並證實了Komica系統的過於陳舊。
    • 大多數島民們的心情從炎上變成炎上（笑），最希望處理的改革也一度從【廢棄版面移置處理辦法】轉移。
  • 當然，還是有極少數（沒跟上討論的）島民，認為Ｍ氏必須為【被退休】出來說明。
    • 雖然Ｍ氏也確實說明了就是，但僅聲明該公告之發布有向其他管理職回報。（資訊來自事情原委#48）
  • 之後，由於聲優廚再度於ＫWiki胡鬧，部分島民關切之重心轉移，且Ａ氏已復職並投入條目復原戰，這次事件中【被退休】的部分暫告落幕。

回應

討論:2017年三月K島管理紛爭事件


半年之後（2017/11）的情況

  • 由於整起事件與前述的廢棄版面移置處理辦法有牽連，故暫放此處。
    • 重申：由於無法中立敘述事件，且事件仍混沌不明，本事件的描述暫時隱藏。可能會永久隱藏。

以下為隱藏的內容
  • 同年2017年十一月中旬，以mymoe.moe作為空間的系列版故障。因未能在七天內修復，暫時依照【廢棄版面移置處理辦法】移動至失效連結處。反mymoe.moe空間的島民雀躍不已。
  • 但mymoe.moe系列版復原後，相關版面又移回原位置，引發要求檢討版面的前述島民不滿；加上於不久前任命的鋼普拉版管理員被認定疑似根本就是廚，不滿的島民將這次事件以及該用人爭議一併找管理員Ａ氏算帳
  • 不滿島民在【mymoe.moe陣亡回報串】與管理員Ａ氏引發激烈筆戰，引發渾沌不明的情勢。最後，一向抗壓性高的管理員Ａ氏其防線崩潰，憤而以公布聲稱難以破解的trip碼的方式，表示不再介入任何管理，放棄身分識別，形同辭職不幹。該被公布的trip碼也馬上被有心人士試用，甚至被廚拿去鬧版
    • 但因為用人爭議懸而未決，認為已經達成目的（但Ａ式的管理職未經解職）的不滿島民仍四處尋找著管理員Ａ氏【本人】可能會出現的串持續鞭笞，要求解決用人爭議
    • 不久後，兩造（疑似含失去身分識別的Ａ氏）於【mymoe.moe陣亡回報串】引發第二波爭論，卻疑似因為廚提及了空穴來風的禁忌傳言，最後該串被發現於11/22整串亡佚。
    • 雖然該串的亡佚引發廚的繼續追殺，但本事件到最後仍然隨時間過去、以下面兩個結果作結：
      • 管理員Ａ氏的用人與管理爭議懸而未決，在換了Trip之後繼續背負著廚給予的罵名回歸管理並緩步改革
      • 廚的目標轉向持續引發爭議的鋼普拉版管理員，有意以【絕對會告贏】為前提蒐集證據以利提告。
