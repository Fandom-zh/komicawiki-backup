ＥＲＲＯＲ！　公開ＰＲＯＸＹ規制中！！[編輯]

  • 出現場合：Futaba 系的貼圖程式內建的基本規制功能，原意是阻擋公開代理伺服器，部份管理員將此功能擴充，用以規制不適當來源主機，常見於 1883 管理的板面上。
  • 狀況說明：管理員有設定一份連線許可的白名單。如果有下列情形之一會出現此錯誤訊息：
    • 使用公開 Proxy
    • 連線不在設定的服務範圍內
    • IP 無法反解
    • 同網段內有人鬧板被限制發文時
  • 解決方法：請到貓館理部板留言（請說明無法正常發文的板面名稱），由管理員更新連線許可清單。

您所使用的連線已被拒絕[編輯]

  • 出現場合：使用 Pixmicat! 的討論板上面。
  • 狀況說明：這功能可能是移植自 2ch 系程式。當管理員開啟 DNSBL (DNS-based Blackhole List) 來源主機檢查功能，而您使用的 IP 剛好在其中任一台 DNSBL 伺服器黑名單上，就會出現此錯誤訊息。
  • 解決方法：
    • 如果您是使用浮動 IP 那就換一個 IP 再連線試看看。
    • 若該討論板使用的是 Pixmicat! 祭典版，則可在名稱中填入"fusianasan" 或 "mokorikomo"（單純使用或配上名稱,如 "名無し@fusianasan"）；這種情況下，您的 IP 或主機名稱會被顯示出來。（非祭典版目前則無此迴避功能）
    • 一勞永逸的做法則是使用較不易被列入黑名單的固定 IP 進行連線，請向您的 ISP 業者洽詢相關事宜。
    • 其實可以對 DNSBL 伺服器要求解鎖，但是需要提出相關證明對 DNSBL 提供單位做解除申請，手續繁複；有的 DNSBL 列表則是一段時間後自動清除，這就不用申請。
  • Pixmicat! 內建檢查的 DNSBL 如下：
sbl-xbl.spamhaus.org
list.dsbl.org
bl.blbl.org
bl.spamcop.net
  • Pixmicat! 祭典版內建檢查的 DNSBL 如下：
Query bad IP from external IP database (0=disabled, 1-8=levels, larger=query more but slower)
Level 1: DSBL チェック（オープンプロクシなどのホストが、登録されています。）
Level 2: BBX チェック（２ちゃんねるで広告爆撃したホストなどが、登録されています。）
Level 3: DNSbl チェック（オープンプロクシなどのホストが、登録されています。）
Level 4: BBQ チェック（２ちゃんねるで悪用されたホストなどが、登録されています。）
Level 5: RBL チェック（ウイルスメールを発信したホストが、登録されています。）
Level 6: ORDB チェック（メールの不正中継したホストが、登録されています。）
Level 7: IRCbl チェック（IRCで爆撃したホストが、登録されています。）
Level 8: 連結標題ToR チェック（ SSL オープンプロクシみたいなの。）

參見[編輯]

  • 踏み台登録確認君，可檢查自己的 IP 是否在 2ch/Pixmicat! DNSBL 檢查規制的範圍內。
  • OpenRBL.org，透過 JavaScript Client 對 40 餘家的 DNSBL 伺服器查詢。
  • Pixmicat! Wiki: DNSBL 黑名單，Pixmicat! 的 Wiki 條目。
